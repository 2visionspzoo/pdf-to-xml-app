1. Header Section (<Naglowek>)
This captures basic invoice metadata.

<Numer>: Invoice number (e.g., "FA/123/2025").
<DataWystawienia>: Issue date (ISO format: YYYY-MM-DD).
<DataSprzedazy>: Sale date (often same as issue date for purchases).
<DataKsiegowania>: Accounting date.
<TypDokumentu>: "ZAKUP" for purchase invoices.
<Kontrahent>: Supplier details.

<NIP>: Tax ID (Polish NIP or EU VAT number).
<Nazwa>: Full name.
<Adres>: Address (street, city, postal code, country).
<KodKraju>: Country code (e.g., "PL").


<FormaPlatnosci>: Payment method (e.g., "przelew" for bank transfer).
<TerminPlatnosci>: Due date.
<Waluta>: Currency (e.g., "PLN", "EUR"); include exchange rate if non-PLN.

2. Line Items Section (<Pozycje>)
Details individual products or services.

Multiple <Pozycja> elements.
<Opis>: Description.
<Ilosc>: Quantity.
<Jednostka>: Unit (e.g., "szt." for pieces).
<CenaNetto>: Net price per unit.
<WartoscNetto>: Total net value.
<StawkaVAT>: VAT rate (e.g., "23" for 23%).
<KwotaVAT>: VAT amount.
<WartoscBrutto>: Gross value.
<KategoriaKsiegowa>: Accounting category (e.g., "401-05" for materials).
<CentrumKosztowe>: Cost center if applicable.

3. VAT Register Section (<RejestrVAT>)
Integrates with Optima's VAT module for tax deduction.

<Typ>: "Rejestr zakupu" for purchases.
<StawkaVAT>: Matches line items.
<Netto>: Total net.
<VAT>: Total VAT (deductible or non-deductible).
<Brutto>: Total gross.
<Odliczalny>: "Tak" if VAT is deductible.
<JPK>: Flags for JPK_V7 compliance (e.g., "TP" for related parties).

4. Payments Section (<Platnosci>)
Optional but useful for scheduling.

<Platnosc>: Per payment.

<Kwota>: Amount.
<DataPlatnosci>: Payment date.
<RachunekBankowy>: Bank account (IBAN format).
<Status>: "rozchód" for outgoing.



5. Additional Elements

<Notatki>: Notes or comments.
<Zalaczniki>: References to attachments (e.g., PDF filenames).
<Korekta>: For corrections, include original invoice reference.
<Wersja>: Schema version (e.g., "2.00" as seen in samples).

For batch imports, enclose in <Dokumenty> with multiple <Dokument> children. The file must be well-formed XML; use tools like XMLSpy or online validators to check.
